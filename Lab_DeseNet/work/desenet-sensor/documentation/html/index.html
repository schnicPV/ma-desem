DeseNet Sensor Project Structure

Following you can find some explanations about the project structure:

Folders starting with IDE

  • ide-qtcreator: Contains Qt Creator project used in simulation environment
  • ide-stm32cubeide: STM32CubeIDE project used to work with the Embedded System hardware

Src Folder Content

The src folder contains source code used in Simulation and Embedded System environment.

  • app: Contains the DeseNet applications and the main file
  • common: Contains packages that are common to the Sensor and Gateway application
    • common/mdw: Contains middleware (MDW) packages:
      • clockwork: Classes for maintaining time
      • containers: Container classes to store and handle data
      • desent: Desent network protocol classes
      • gui: Platform independent graphical classes
      • pool: Contains static object pool class to omit usage of heap
      • trace: Package to output message text via UART, USB or other peripherals

    • common/platform: This projects supports multiple platforms. One is the real hardware and one represents a simulation environment simulating the hardware and the network communication:

      • nucleo-stm32l476rg: Represents the real hardware, hence the Nucleo development board.
      • qt-meshsim: Represents the simulation environment using the MeshSimulator (running on Windows, macOS and Linux).
      • platform-common: Contains packages and files which are common for all platforms.

      Each platform has a board, desent and mcu package:

      • board: Contains board specific code (buttons, accelerometer, etc.)
      • desent: The platform specific transeiver driver and time slot manager
      • mcu: Microcontroller and peripheral specific code (UART, SPI, etc.)
  • xf: Contains the sources of the XF (eXecution Framework).
